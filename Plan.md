## שלב נוכחי: פיתוח אפליקציה

### רקע — למה עוברים לאפליקציה
- המודל הטוב ביותר (V3: **HistGradientBoosting, MAE=0.263, R²=0.603**) הגיע לתקרת ביצועים
- אומתה תקרה: 19 מודלים שונים נוסו (כולל LightGBM, CatBoost, MLP, KNN, SVR, Stacking) — שיפור של 2.6% בלבד מעל RF tuned
- התקרה = אירועים בלתי-צפויים (drift), לא בחירת מודל — כל המודלים נכשלים בשאגת הארי באותו אופן
- הערך הבא נמצא בהנגשה ובממשק, לא בשיפור MAE

## 🗓️ מפת דרכים — מפגשי המנטור הנותרים (מעודכן 2026-05-09)

### מפגש 3 — שכבת GenAI והסברתיות
**מטרה:** להוסיף שכבת AI שמסבירה תחזיות בעברית טבעית.
- חיבור ל-Claude API (claude-sonnet-4-6)
- ממשק שאילתות בעברית: "מה הרייטינג הצפוי לקבינט שישי השבוע?"
- הסבר אוטומטי לכל תחזית: "החיזוי גבוה כי בשבועות האחרונים הרצועה הזו נתנה רייטינג ממוצע של X"
- SHAP/LIME — feature importance ברמת תחזית בודדת
- 📤 **תוצר:** ממשק שאלות-תשובות עובד + הסברים אוטומטיים

### מפגש 4-5 — איטרציות מוצר
**מטרה:** הפיכת ה-MVP למוצר אמיתי שמישהו יכול להשתמש בו.
- בדיקת משתמש (user testing) — להעביר את האפליקציה למישהו אמיתי (המנטור? איש תוכן?)
- שיפור UX לפי פידבק
- הוספת פיצ'רים שעלו מהשימוש בפועל
- תיקון באגים, ליטוש עיצוב, אופטימיזציית מהירות
- 📤 **תוצר:** אפליקציה יציבה ומלוטשת, מוכנה לפריסה

### מפגש 6 — פריסה לתיק עבודות
**מטרה:** להוציא את הפרוייקט החוצה כך שיהיה אפשר להראות ולמכור.
- פריסה בענן עם URL ציבורי
- README באנגלית שמציג את הפרוייקט (לעיני מעסיקים בינלאומיים)
- שני repos: private (קוד מלא + נתונים), public (גרסה מסונטזת לתצוגה)
- סרטון demo / צילומי מסך
- פוסט בלינקדאין
- 📤 **תוצר:** פרוייקט חי + URL + פרסום בלינקדאין

## החלטות ארכיטקטורה ✅ (2026-05-09)

### Persona ראשי ל-MVP: מנהל/ת תוכן
- **שימוש:** יומי-שבועי, תכנון לוח שידורים
- **אופק תכנון:** חודש קדימה (ראשי) + שבוע (zoom-in)
- **משתמשים:** 1-2 — אין צורך באימות / multi-user
- **personas נוספים שזוהו** (לא ב-MVP): מכירות-פרסום, שיווק, הנהלה/מחקר

### Stack טכני
| רכיב | החלטה |
|---|---|
| Frontend | **Streamlit** + multi-page |
| Visualization | **Plotly** (אינטראקטיבי, hover) |
| מקור חיזויים | קובץ מוכן (`predictions_v2.xlsx`) — לא מודל חי |
| Deployment | **Streamlit Community Cloud** (GitHub repo) |

### 4 מסכים ל-MVP
1. **לוח חודשי** — תצוגה ראשית, חיזויים על פני 30 יום
2. **השבוע** — zoom-in שבועי עם פירוט לפי רצועה
3. **כרטיס תוכנית** — drill-down: היסטוריה, ממוצעים, אירועים
4. **השוואה** — בחירת 2-3 תוכניות והשוואת חיזויים
