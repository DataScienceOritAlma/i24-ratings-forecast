### גישה למוצר אמיתי (מפגש 6 ואחרי): FastAPI + React

| רכיב | טכנולוגיה מומלצת | למה |
|---|---|---|
| **Backend** | FastAPI (Python) על Render/Railway | מהיר, חינם להתחלה |
| **Frontend** | React + Tailwind | סטנדרט תעשייה, מרשים בראיונות |
| **DB** | Supabase (PostgreSQL) | חינם עד 500MB, auth מובנה |
| **Auth** | Supabase Auth או Auth0 | login עם Google/אימייל |
| **Deploy** | Vercel (frontend) + Render (backend) | חינם, אוטומטי מ-GitHub |

**ההבדל הקריטי:** אפליקציה אמיתית מצליחה לטפל ב-100 משתמשים בו-זמנית, לאבטח דאטה, ולשמור היסטוריה. Streamlit לא יכולה את זה.

## 📁 GitHub ותיק עבודות — איך להעלות בלי לחשוף

### האסטרטגיה המומלצת: **שני repos**

#### Repo פרטי (הכל)
- כל הקוד + הנתונים האמיתיים של i24 + מודלים שמורים
- private — רק את (ואולי המנטור) רואה
- כאן את עובדת באמת

#### Repo ציבורי (לראווה)
- אותו קוד, אבל:
  - הנתונים מוחלפים בנתונים סינתטיים (random + תבניות דומות)
  - שמות תוכניות אמיתיים מוסתרים ("Program A", "Program B")
  - README מצוין באנגלית
  - סרטון demo מוטמע
  - URL לאפליקציה החיה
- public — מה ששולחים בראיונות

### איך להעלות פרוייקט ישן ל-GitHub
```bash
cd "תיקיית הפרוייקט הישן"
git init
git add .
git commit -m "Initial commit"
gh repo create my-old-project --private --source=. --push
```
*(צריך התקנה של `gh` — GitHub CLI)*

**טיפ חשוב:** הוסיפי `.gitignore` לפני שמעלים, אחרת תעלי בטעות `__pycache__`, `.env` וקבצים גדולים.

### דוגמה ל-`.gitignore` בסיסי
```

*.pyc

.streamlit/secrets.toml
*.xlsx
*.csv
data/
```

## 📊 נתוני דיגיטל — האם להוסיף?

### האם נתוני דיגיטל משפיעים על רייטינג?
**כן, מאוד.** מחקרים מהענף מראים:
- **כתבה ויראלית באתר** → אנשים פותחים את הטלוויזיה לראות עוד → רייטינג עולה ב-5-15%
- **Push notifications של ברייקינג** → קפיצה מיידית ברייטינג בשעה הבאה
- **Engagement ברשתות חברתיות** → אינדיקטור מקדים לרייטינג של אותו ערב

### מה שווה לבקש מ-i24
1. **תנועה יומית לאתר** (sessions, page views)
2. **Push notifications** (כמה נשלחו, על איזה נושא, מתי)
3. **Top stories של היום** (קטגוריאלי: ביטחון/פוליטיקה/כלכלה)
4. **Engagement ברשתות** (likes, shares, comments בפוסטים)
5. **App active users** (אם קיימת אפליקציה)

## 📺 דקות פרסום לכל תוכנית — איך זה ישנה את הפרוייקט

### תרומה צפויה
- שיפור MAE ב-5-15% (אם הקשר חזק)
- הסבר טוב יותר של "למה הצופים עזבו באמצע התוכנית"
- פיצ'ר חדש: "כמה פרסומות אופטימליות לפני שמאבדים צופים?"

### מה זה משנה במודל
- פיצ'ר חדש: `ad_minutes_per_program`
- פיצ'ר נגזר: `ad_density` (דקות פרסום ÷ אורך תוכנית)
- פיצ'ר נוסף: `ad_breaks_count` (כמה הפסקות פרסום)

### ערך עסקי גדול
המודל יוכל לענות על שאלה ש-i24 משלמת עליה כסף:
> "כמה פרסום לשים בתוכנית הזו בלי לאבד צופים?"

זה הופך אותך ממנתחת דאטה ליועצת אסטרטגית.
